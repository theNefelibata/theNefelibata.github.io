    版本管理和Git的设计

    版本管理的需求

    • 时间机器：
      • 全量方案：保存项目的每一个版本的完整信息
      • 增量方案：保存每一版的修改，版本一+补丁=版本二
    优点 缺点 使用场景
    全量方案 简单、快速 重复存储，浪费空间 代码，文本
    增量方案 存储空间小 需要运算，降低了性能，增加了运算时间 视频，大型二进制文件


    系统记录了每一个修改版本的完整信息，存储在.git中

    常用Git命令

    • git init
      • git初始化命令
      • 将一个文件夹初始化为git仓库
    • git add filename
      • git add ./* 将所有文件添加到暂存区
      • 将filename文件添加到暂存区
    • git status
      • 查看暂存区状态
    • git reset HEAD--a.txt
      • 将文件移出暂存区
    • git cat-file -p SHA-1
      • 查看SHA-1文件内容
    • git ls-files --stage
      • 查看暂存区内容
    • git commit -m ‘ ‘
      • 将暂存区文件提交到git仓库
    • git branch name
      • 创建一个名为name的分支
    • git checkout name
      • 切换到某个分支
      • 回到某个历史分支git checkout SHA-1
      • git checkout -b name 创建一个分支并切换到分支
    • git merge branch-name
      • 将分支branch-name合并到当前分支
    • 合并失败：
      • git merge --abort放弃合并
      • 手动解决冲突后再提交，然后合并
    • git log
      • 查看git日志
    • git pull (origin main)
      • 将远程仓库的内容合并到本地仓库
    • git push
      • 将本地仓库的内容push到远程仓库
    • git stash
      • 将当前修改放入一个列表
        • git stash list
        • git stash apply
      • git stash show/drop/clear
